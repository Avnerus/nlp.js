Edit the chatbot's intents and responses. The JSON structure matches the standard NLP.js corpus format. Each intent contains utterances (phrases that trigger it) and answers (responses to give).

What is a Corpus?

A corpus is a JSON file that defines what your chatbot knows. It contains:

  • name - The corpus name
  • locale - Language locale (e.g., "en-US")
  • entities - Named entities that can be extracted from user input
  • data - Array of intents

Basic Structure

Each intent has three parts:

  • intent - A unique name for this intent
  • utterances - Array of phrases that trigger it
  • answers - Array of possible responses
{
  "intent": "greetings.hello",
  "utterances": ["hello", "hi", "hey"],
  "answers": ["Hello!", "Hi there!", "Greetings!"]
}

Simple Answers

Just list text responses - the chatbot will pick one at random:

{
  "intent": "agent.age",
  "utterances": ["how old are you", "your age"],
  "answers": ["I'm very young", "I was created recently"]
}

Advanced Usage

Entities

Entities are named patterns that extract specific information from user utterances. For example, you can extract a username, date, or any custom pattern.

An entity has:

  • trim - Rules to extract the value from the utterance
  • options - Predefined valid options for this entity

In the template corpus, the username entity is defined to extract the user's name from phrases like "I'm Avner" or "My name is Avner".

Slot Filling

Slot filling allows you to ask follow-up questions when required entities aren't provided. This is defined in the intent's slotFilling field.

When the user triggers an intent but doesn't provide the required entity, the chatbot will ask the question defined in slotFilling before proceeding.

Example: The greetings.hello intent uses slot filling to ask for the username if not provided:

{
  "intent": "greetings.hello",
  "utterances": ["hello", "hi", "howdy"],
  "answers": ["Hey {{ username }}! Nice to meet you."],
  "slotFilling": {
    "username": {
      "mandatory": true,
      "question": "I don't think we've met! What is your name?"
    }
  }
}

The {{ username }} will be replaced with the extracted entity value from the slot filling process.

Conditional Answers

Use the opts field for conditional responses based on entity values or context:

Example: The user.likeagent intent gives different responses based on whether the user is "avner" (the creator):

{
  "intent": "user.likeagent",
  "utterances": ["I like you", "I really like you"],
  "answers": [
    {
      "answer": "Of course, because you created me.",
      "opts": "entities.username.option === 'avner'"
    },
    {
      "answer": "Likewise!",
      "opts": "entities.username.option !== 'avner'"
    }
  ]
}

The opts expression is evaluated at runtime - if true, that answer is used.

Template Variables

Use {{ variable }} to insert dynamic values like slot-filled entities:

Example: The greetings.hello intent uses {{ username }} to greet by name:

{
  "intent": "greetings.hello",
  "utterances": ["hello", "hi", "howdy"],
  "answers": [
    "{{ username === 'Avner' ? 'My creator! (along with fattybear). Honored to meet you.' : 'Greetings ' + username + '! Nice to meet you!' }}"
  ],
  "slotFilling": {
    "username": {
      "mandatory": true,
      "question": "I don't think we've met! What is your name?"
    }
  }
}

You can also use JavaScript expressions in the template variables for conditional logic.
