What is a Corpus?

A corpus is a JSON file that defines what your chatbot knows. It contains intents - groups of phrases (utterances) that trigger specific responses (answers).

Basic Structure

Each intent has three parts:

  • intent - A unique name for this intent
  • utterances - Array of phrases that trigger it
  • answers - Array of possible responses
{
  "intent": "greetings.hello",
  "utterances": ["hello", "hi", "hey"],
  "answers": ["Hello!", "Hi there!", "Greetings!"]
}

Simple Answers

Just list text responses - the chatbot will pick one at random:

{
  "intent": "agent.age",
  "utterances": ["how old are you", "your age"],
  "answers": ["I'm very young", "I was created recently"]
}

Conditional Answers

Use the opts field for conditional responses based on context:

{
  "intent": "agent.whereami",
  "utterances": ["where am i"],
  "answers": [
    { "answer": "You're on mobile", "opts": "channel==='telegram'" },
    { "answer": "You're on desktop", "opts": "channel==='web'" }
  ]
}

Template Variables

Use {{ variable }} to insert dynamic values:

{
  "intent": "agent.whereami",
  "utterances": ["where am i"],
  "answers": ["You're talking from {{ channel }}"]
}

Common Intents to Include

Start with these fundamental intents:

// Agent identity
- agent.acquaintance: "who are you", "what are you"
- agent.age: "how old", "your age"
- agent.whereami: "where am i", "where are we"

// Greetings
- greetings.hello: "hello", "hi", "hey"
- greetings.bye: "goodbye", "bye", "see you"

// Help/Support
- help: "help", "what can you do", "how do I"

// Small talk
- yes: "yes", "yeah", "yep"
- no: "no", "nope", "nah"

Full Corpus Structure

Your corpus must be wrapped in an object with name, locale, and data fields:

{
  "name": "Corpus",
  "locale": "en-US",
  "data": [
    {
      "intent": "intent.name",
      "utterances": ["phrase 1", "phrase 2"],
      "answers": ["response 1", "response 2"]
    }
  ]
}
